Exploring the axiomatic semantics of array access

Charlie Harding

The following is based on the axiomatic semantic rules for the While language.

Assignment

The following is the rule proposed for assignment as per Nielson and Nielson.

ASSNPxax := aP

Example

ASSN(y = 12) ? 5 : 7 = 5x := (y = 12) ? 5 : 7x = 5 CONS
5y = 12
7otherwise
 = 5x := (y = 12) ? 5 : 7x = 5 CONS y = 12x := (y = 12) ? 5 : 7x = 5

Note that the CONS rule has been split in two, to make clear the derivation process.

Array Assignment

I propose the following rule for array assignment:

ARRAPxa₁a₂x[a₁] := a₂P

where arr[x] resolves to arrx, which in turn could resolve to "arr;" concatenated with the string representation of x.

Example

ARRA x = y 5 = 5arr[y] := 5x = y arry = 5 COND x = y 5 = 5arr[y] := 5arrx = 5 CONS x = yarr[y] := 5arr[x] = 5

My concern with this approach, however, is that the onus of converting array accesses has to be on the right-hand side, rather than the left-hand size. It also seems a bit strange that the evaluation of arr[x] occurs as the consequent of an evaluation x, but I presume this is because the condition involves a lot of syntax.
